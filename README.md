# SMAI Repository

This repository contains coursework, notebook submissions, and supporting material for the **Statistical Methods in Artificial Intelligence (SMAI)** course.

The workspace is organized around assignment submissions, daily class exercises, and reference material. Most work is implemented in Jupyter notebooks, with a small number of helper scripts and generated assets stored alongside the relevant notebooks.

## Contents

- [Assignments](#assignments)
- [Daily Class Work](#daily-class-work)
- [Course Material](#course-material)
- [Running The Notebooks](#running-the-notebooks)

## Assignments

### Assignment 1

`Assignment_1/` contains the submission material for the first assignment:

- `Q1_Bias_Variance_Regularisation.ipynb` - bias-variance and regularisation analysis.
- `Q2.ipynb` - data normalisation and KNN classification.
- `Q3.ipynb` - linear regression under noise and outliers.
- `supplementary_code_q4.py` - helper code used for the Q4 visualisation.
- `dataset_cache/` - cached dataset files used by the notebooks.
- `plots/` - generated plots and figures.

### Assignment 2

`Assignment_2/` contains the second assignment notebook work and supporting files:

- `q1_trial.ipynb` - primary notebook draft.
- `q1_trial copy.ipynb` - alternate notebook draft.
- `SMAI_A2.pdf` - assignment statement.
- `Dataset/` - dataset preparation scripts and extracted sample folders.

## Daily Class Work

`Daily_Class_Works/` contains notebook-based exercises and short implementations covering:

- KNN and linear classifiers
- feature transformation and performance metrics
- linear regression and regularised regression
- PCA and dimensionality reduction
- Bayesian methods, logistic regression, SVMs, perceptron, MLPs, backpropagation, CNNs, and RNNs

## Course Material

- `Course_Slides/` - lecture slides and other course reference material.

## Running The Notebooks

1. Open the desired notebook in VS Code or Jupyter.
2. Select the Python environment for the workspace.
3. Run cells in order to reproduce the results.
4. Keep cached datasets, plots, and helper scripts in place if a notebook depends on them.

## Notes

- The repository includes both final submission notebooks and exploratory trial notebooks.
- Generated outputs such as cached data and plots are stored inside the assignment folders.
- When adding new work, keep related helper scripts and outputs close to the notebook that uses them.
